> **Skill类型**: 文档生成类
> **触发词**: 生成信息图、产品一页纸、竞品对比、做信息图
> **核心承诺**: 设计风格100%一致、品牌色准确、内容不编造

## 触发条件

### ✅DO: 应该触发

用户说以下任一内容时，**必须**触发此Skill：

触发词：「生成信息图」、「做信息图」、「信息图」、「产品一页纸」、「产品介绍页」、「一页纸」、「竞品对比」、「对比分析」、「竞品分析」、「帮我做XXX的信息图」、「XXX的产品一页纸」

## 产出物定义 📦

### 产品一页纸HTML
**文件路径**: `infographic/{产品名}-产品一页纸/{产品名}-产品一页纸.html`

**必须包含的板块**（按顺序）：
1. ✅ 顶部装饰条（6px渐变）
2. ✅ 头部区域（品牌logo + 产品名称 + 口号）
3. ✅ 核心卖点POD（2×2网格，4个卖点）
4. ✅ 产品信息（规格/使用方法/适用人群/禁忌人群，4列网格）
5. ✅ 注意事项（独立板块，2列列表，⚠图标）
6. ✅ 三种香型（如有，3列网格，绿/橙/黄渐变卡片）
7. ✅ 使用场景（3列网格）
8. ✅ 产品口播（浅橙红底+品牌色文字）
9. ✅ 常见问题FAQ（2列网格）
10. ✅ 底部区域（深色底+logo+免责声明）

**设计参数（定值法）**：
- 品牌色: `#E85A24`（PANTONE 1645C，✅DO NOT使用`#87032`等错误色值）
- Logo: 相对路径`logo.png`，CSS:`filter: brightness(0) invert(1)`（变白色）
- 背景: `#faf8f2`（宣纸暖白）
- 字体: 标题`Noto Serif SC`，正文`Noto Sans SC`
- 卡片: 白底+`#e0d5c0`边框+6px圆角+轻微阴影
- ✅DO NOT使用冷色调（蓝/冷绿/冷灰）

### 竞品对比HTML
**文件路径**: `infographic/{产品名}-竞品对比/{产品名}-竞品对比.html`

**必须包含的板块**：
1. ✅ 竞品定位（4张卡片，各用不同暖调背景）
2. ✅ 核心参数速查（4张参数卡片，通用化，不叫"技术路径"）
3. ✅ 多维度综合对比（雷达图三列排布：左右说明卡片 + 中间雷达图）
4. ✅ 核心结论（浅橙红底 + 品牌色标题）

**竞品配色（定值法）**：
- 蔡氏福宁: 品牌橙红 `#E85A24`
- 竞品1: 暖钢灰蓝 `#4a6b7a`
- 竞品2: 暖橄榄绿 `#5a8a5a`
- 竞品3: 暖铜棕 `#a07030`

### 中间文件（markdown）
**文件路径**: `infographic/{产品名}-{类型}/{产品名}_{类型}内容.md`

**用途**: 方便用户微调文案，再从模板生成HTML

## 工作流程

### 重要：使用模板生成HTML（不是重新设计）

**✅DO: 基于模板修改内容**
- ✅DO NOT重新写HTML结构——会导致设计风格走样（v5版本教训）
- ✅DO: 复制`assets/`目录中的模板到目标目录
- ✅DO: 读取markdown文件，提取内容填充模板占位符

**模板保证出品一致性**：
- 设计风格100%与v3版本一致
- CSS变量定义完全一致
- 板块顺序和布局完全一致

### 第一步：提取内容生成markdown

#### ☑SHOULD: 读取brief文件

**✅DO NOT编造内容**——所有内容必须从brief中提取。

**✅DO NOT上下文污染**——只使用当前产品的brief，不使用其他产品的信息。

1. **用户提供参考**：
   - 如果用户提供了brief文件路径，使用`Read`工具读取
   - 如果用户提供了IMA链接，使用`mcp__ima-mcp__fetch_media_content`读取
   - ✅DO: 读取完成后，确认"我只读取了{当前产品}的brief"
   - ✅DO NOT: 同时保留上一个产品的brief内容在working context中

2. **用户未提供参考**：
   - ✅DO: 询问用户提供brief文件
   - ✅DO NOT: 自己编造产品信息

#### ☑SHOULD: 提取关键信息

按照`references/brief-structure.md`中的标准结构提取：

1. **产品层**：
   - 产品名称、规格、SKU列表
   - 核心卖点POD（特征/优势/利益/vs竞品/RTB证据）
   - 使用方法
   - 适用人群（从brief中提取，写关键内容）
   - 禁忌人群（从brief中提取，写关键内容）
   - 注意事项（从brief中提取，写关键内容，6-7条）
   
2. **心智层**：
   - 产品口号（slogan）
   - 口播文案（如有）
   
3. **市场层**：
   - 竞品信息（名称/价格/定位/技术路径）
   - 价格定位
   
4. **利益层**：
   - 目标人群
   - 使用场景
   - 核心价值

#### ☑SHOULD: 生成markdown文件

**✅DO: 严格按照以下规范生成markdown**：

1. **文件名**: `{产品名}_信息图内容.md`（或`{产品名}_竞品对比内容.md`）
2. **位置**: 与HTML信息图同目录（`infographic/{产品名}-{类型}/`）
3. **内容结构**: 严格按照`references/brief-structure.md`中的"信息图内容结构"
4. **✅DO NOT编造**: 如果brief中没有某个信息，写`[待补充]`而不是自己编
5. **✅DO NOT上下文污染**: 生成前，逐条确认内容来自当前产品的brief，不是其他产品

**防污染检查清单**（生成markdown后，必须执行）：
- [ ] 产品名称正确（不是其他产品）
- [ ] 背书人/代言人正确（如果有，必须来自当前产品brief）
- [ ] 技术名称正确（不是其他产品的技术）
- [ ] 古籍引用正确（如果有，必须来自当前产品brief）
- [ ] 竞品名称正确（来自当前产品brief，不是其他产品的竞品）

#### ☑SHOULD: 展示markdown文件给用户

1. 使用`present_files`展示markdown文件
2. 询问用户："文案内容是否需要微调？"
3. 如果用户提出修改，优先修改markdown文件（方便微调）
4. 修改完成后，再进入下一步
5. **⚠️ 雷达图确认（竞品对比专用）**：
   - 展示markdown后，额外询问用户：
     "雷达图维度和各品牌打分排名是否准确？请确认：\n     - 维度列表：{维度1}、{维度2}...\n     - 各品牌各维度排名：{品牌名} > {竞品1} > ..."
   - ✅DO：用户确认后，再写入HTML
   - ✅DO NOT：自己填打分数据（高频错误）

### 第二步：基于模板生成HTML（关键步骤）

#### ☑SHOULD: 复制模板到目标目录

**✅DO: 使用Bash工具复制模板**（不是重新写HTML）：

```bash
# 产品一页纸
cp assets/product-one-pager-template.html infographic/{产品名}-产品一页纸/{产品名}-产品一页纸.html

# 竞品对比
cp assets/competitor-comparison-template.html infographic/{产品名}-竞品对比/{产品名}-竞品对比.html
```

#### ☑SHOULD: 复制logo到HTML目录

**✅DO: logo文件已内置在skill的assets/目录中**（v2.2更新：不再使用硬编码本地路径）

```bash
# 从skill的assets目录复制logo（通用路径，适用于所有用户）
cp ~/.workbuddy/skills/product-infographic-generator/assets/logo.png infographic/{产品名}-{类型}/logo.png
```

**⚠️ 如果上述路径找不到logo**：
- 原因1：skill未正确安装到`~/.workbuddy/skills/`
- 原因2：assets目录被删除
- 解决：重新安装skill，或手动将蔡氏福宁logo.png放到上述路径

#### ☑SHOULD: 读取markdown，填充模板

1. 使用`Read`工具读取markdown文件
2. 解析markdown结构，提取各部分内容
3. **✅关键步骤：先统计各板块实际数量**
   - 统计POD数量（通常4个）
   - 统计香型数量（0/2/3种）
   - 统计使用场景数量（3/4/5个）
   - 统计FAQ数量（4/5/6/7个）
   - 统计注意事项数量（6-9条）
   - ⚠️ 如果数量与模板默认值不同，**必须调整HTML结构**
4. 使用`Edit`工具替换HTML模板中的占位符：
   - `{产品名称}` → 产品名称
   - `{产品口号}` → 口号
   - `{POD1标题}` → POD1标题
   - ...（完整占位符列表见模板文件）

#### ☑SHOULD: 动态板块匹配（⚠️核心步骤，不可跳过）

**模板中有以下动态板块，必须根据markdown实际内容调整**：

| 板块 | 模板默认 | 动态范围 | 调整方式 |
|:-----|:---------|:---------|:---------|
| 香型 | 3列 | 0(删除整个section) / 2 / 3 | 删除section或删card+改grid |
| **使用场景** | **3列** | **3 / 4 / 5** | **增删card + 改grid列数** |
| FAQ | 6个 | 4 / 5 / 6 / 7 | 增删faq-item |
| 注意事项 | 2列布局 | 任意条数 | 只需增删li，不改结构 |

**具体操作方法**：

**场景数量调整示例**（休足贴有4个场景）：
```css
/* CSS修改 */
.scene-grid {
  grid-template-columns: repeat(4, 1fr);  /* 从repeat(3)改为repeat(4) */
}
```
```html
<!-- HTML：追加第4个场景卡片 -->
<div class="scene-card">
  <div class="scene-icon">{场景4图标}</div>
  <div class="scene-title">{场景4标题}</div>
  <div class="scene-desc">{场景4描述}</div>
</div>
```

**无香型产品处理**：
```html
<!-- 删除整个scent-section（从<div class="scent-section">到</div>） -->
```

**⚠️ 真实事件（2026-07-06同事反馈）**：
- 休足贴brief中有4个使用场景
- 模板默认只有3个场景卡片
- 导致"暴走后的酒店回血"场景被漏掉
- **教训：必须先统计数量，再决定是否需要调整模板结构**

#### ✅DO: 关键设计规范检查（定型法）

**✅DO NOT跳过此检查**——品牌色错误是高频问题。

1. **品牌色检查**：
   - ✅DO: 在CSS中定义`:root { --brand: #E85A24; }`
   - ✅DO NOT: 使用`#87032`（错误色值，v3之前出现过）
   - ✅DO NOT: 使用`#FF0000`（纯红色，不是品牌色）

2. **Logo检查**：
   - ✅DO: 使用相对路径`<img src="logo.png" alt="蔡氏福宁" class="header-logo" />`
   - ✅DO: 确认logo文件已复制到HTML同目录
   - ✅DO: 在浏览器中打开HTML，检查logo是否显示

3. **配色检查**：
   - ✅DO: 所有辅助色都用暖调（见`references/brand_guidelines.md`）
   - ✅DO NOT: 使用冷蓝色（如`#4a90d9`）
   - ✅DO NOT: 使用冷绿色（如`#2ecc71`）

4. **内容检查**：
   - ✅DO: 所有内容从brief中提取
   - ✅DO NOT: 编造brief中不存在的信息
   - ✅DO: 品牌名用"蔡氏福宁"，✅DO NOT用拼音"CAISHIFUNING"

#### ☑SHOULD: 预览HTML文件

1. 使用`present_files`展示HTML文件
2. 在浏览器中打开，检查效果
3. 重点检查：
   - 品牌色是否正确显示
   - Logo是否正常显示
   - 配色是否和谐（全部暖调）
   - 内容是否准确（无编造）

#### ☑SHOULD: 根据反馈调整

1. 如果用户提出修改意见：
   - **优先修改markdown文件**（方便微调）
   - 修改完成后，重新从模板生成HTML（不要手动改HTML）
   
2. ✅DO NOT: 直接手动修改HTML文件——会导致下次生成时风格不一致

## Poka-Yoke防误层

### 接触法：物理验证

1. **✅DO: 必须先Read模板文件，再复制修改**
   - 禁止：不Read就直接Write HTML（会导致模板结构丢失）
   
2. **✅DO: 必须先Read brief文件，再生成markdown**
   - 禁止：不Read就自己编内容
   
3. **✅DO: 生成HTML后，必须在浏览器中打开检查**
   - 禁止：只生成不检查

### 定值法：固定参数

1. **品牌色固定**: `#E85A24`（PANTONE 1645C）
   - ✅DO NOT: 使用其他色值
   - 检查方法：在CSS中搜索`#`，确认所有颜色都在暖色相环内
   
2. **Logo处理固定**: 相对路径+filter变白色
   - ✅DO NOT: 使用绝对路径（跨设备会失效）
   - ✅DO NOT: 使用base64嵌入（MIME类型容易拼错）
   
3. **触发词固定**: 至少3个
   - "生成信息图"、"产品一页纸"、"竞品对比"
   - ✅DO: 在description中明确定义

### 动作步序法：步骤强依赖

1. **第一步完成，才能进入第二步**：
   ```
   读取brief → 生成markdown → [用户确认] → 复制模板 → 填充内容 → 预览
   ```
   - ✅DO NOT: 跳过markdown直接生成HTML
   - ✅DO NOT: 不用户确认就直接生成最终HTML
   
2. **修改文案时，先改markdown再生成HTML**：
   - ✅DO NOT: 直接手动改HTML（会导致模板结构丢失）
   - ✅DO: 改markdown → 重新从模板生成HTML

## 熔断器

### 失败=停止并报告

1. **如果brief文件不存在**：
   - ✅DO: 停止并询问用户："请提供产品brief文件"
   - ✅DO NOT: 自己编造产品信息
   
2. **如果品牌色值不是`#E85A24`**：
   - ✅DO: 停止并更正
   - 参考：`references/brand_guidelines.md`
   
3. **如果logo无法显示**：
   - ✅DO: 尝试方案A（相对路径）
   - ✅DO: 如果方案A不行，尝试方案B（复制logo到HTML同目录）
   - ✅DO: 如果都不行，停止并报告用户
   
4. **如果配色不协调（出现冷色调）**：
   - ✅DO: 停止并参考`references/brand_guidelines.md`修正
   
5. **如果用户要求导出PNG**：
   - ✅DO: 告知用户"导出功能已删除，请用浏览器另存为"
   - ✅DO NOT: 添加html2canvas导出功能（会导致logo消失）

6. **如果竞品对比的markdown中间文件不存在**：
   - ✅DO: 停止并先生成markdown文件，展示给用户确认
   - ✅DO NOT: 跳过markdown直接生成HTML
   - 原因：竞品对比的数据（雷达图打分、各品牌参数）需要用户确认后才能生成HTML

7. **如果检测到内容污染（当前产品出现了其他产品的专属信息）**：
   - ✅DO: 停止并逐条核对所有内容是否来自当前产品的brief
   - ✅DO NOT: 沿用上一个产品的内容而不检查
   - 常见污染：背书人/代言人、产品规格、技术名称、古籍引用
   - 原因：连续生成多个产品时，容易把上一个产品的内容混到当前产品中（三伏贴→休足贴污染事件）

8. **如果检测到信息遗漏（brief中有但生成文件没有）**：
   - ✅DO: 逐条核对brief中的列表项数量
   - ✅DO: 特别注意"注意事项""FAQ""适用人群"等列表型内容
   - 原因：提取时容易漏掉最后几条（休足贴注意事项少写1条事件）

9. **如果检测到信息理解错误（技术参数被改写）**：
   - ✅DO: 引用brief原文，不改写技术参数
   - ✅DO: 特别注意数字、单位、范围（如：pH≈8不能写成pH=8.0）
   - 原因：AI容易"规范化"技术参数，导致精度丢失

10. **如果检测到模板结构与内容数量不匹配**：
    - ✅DO: 停止并统计markdown中各板块的实际数量
    - ✅DO: 调整HTML结构以匹配实际内容数量
    - ✅DO NOT: 直接用默认模板填充，导致内容丢失
    - 常见不匹配：
      - 使用场景：模板3列 vs 实际4个（休足贴"暴走后的酒店回血"被漏掉）
      - FAQ：模板6个 vs 实际5个或7个
      - 香型：模板3列 vs 实际0种（无香型产品）或2种
      - 竞品卡片：模板4张 vs 实际2-3个竞品
    - 原因：模板是参考蒸汽眼罩制作的，其他产品的板块数量可能不同
    - 修复方法：见"动态板块匹配"章节的操作指南

## Alternatives Considered

### 方案A: 重新设计HTML each time
- **选择**: ❌未选择
- **理由**: 
  - 会导致设计风格不一致（v5版本教训）
  - 每次都要重新调整配色、字体、间距
  - 出品稳定性差
  
### 方案B: 基于模板修改内容（✅已选择）
- **选择**: ✅已选择
- **理由**: 
  - 设计风格100%一致
  - 只需修改内容，不需要调整样式
  - 出品稳定性有保障
  
### 方案C: 使用即梦AI生成信息图
- **选择**: ❌未选择
- **理由**: 
  - 文字错误率高
  - 信息密度低
  - 无法精确控制设计风格
  
### 方案D: 添加html2canvas导出PNG功能
- **选择**: ❌未选择（初始版本有，后删除）
- **理由**: 
  - logo在导出时会消失（本地文件无法被html2canvas渲染）
  - 用户可以用浏览器"另存为"替代
  
### 方案E: 竞品配色用冷色调区分
- **选择**: ❌未选择
- **理由**: 
  - 与品牌暖调设计系统不和谐
  - 所有竞品色都应在暖色相环内

## 信息准确性保障（完整性检查）

### 信息不准确的6种可能性

#### 1. 编造内容（无中生有）
**表现**：brief中没有的信息，AI自己加上去
**例子**：brief没提"创始于1999年"，AI自己加上
**防护**：✅DO NOT编造 + 写`[待补充]`

#### 2. 上下文污染（用错来源）
**表现**：把A产品的信息用到B产品
**例子**：休足贴出现三伏贴的"郭昭明"
**防护**：防污染检查清单（5项必查）

#### 3. 信息遗漏（该写没写）
**表现**：brief中有，但AI漏掉了
**例子**：休足贴注意事项brief有9条，生成只有8条（缺小儿足部注意事项）
**防护**：逐条核对brief中的列表项数量

#### 4. 信息理解错误（理解偏差）
**表现**：brief中有，但AI理解错了
**例子**：brief说"pH≈8弱碱性"，AI写成"pH=8.0"
**防护**：引用brief原文，不改写技术参数

#### 5. 信息提取错误（找错位置）
**表现**：把A板块的内容填到B板块
**例子**：把"注意事项"填到"适用人群"
**防护**：按brief-structure.md的结构逐板块提取

#### 6. 主观加工（添加个人理解）
**表现**：brief是客观描述，AI加了主观评价
**例子**：brief说"竞品无中医穴位概念"，AI写成"竞品技术落后"
**防护**：竞品对比只做客观参数对比，不加主观评判

## 重要提醒

### 1. 品牌色必须准确（定值法）

**错误示例**：
- ❌ `#87032`（色值错误，v3之前出现过）
- ❌ `#FF0000`（纯红色，不是品牌色）
- ❌ `#E85A24`但写成了`#E85A24`（拼写错误）

**正确示例**：
- ✅ `#E85A24`（PANTONE 1645C 亮橙红）

**检查方法**：
- 在CSS中定义 `:root { --brand: #E85A24; }`
- 所有使用品牌色的地方用 `var(--brand)`，不要硬编码

### 2. Logo必须显示（接触法）

**检查清单**：
1. ✅DO: 确认logo文件已复制到HTML同目录
2. ✅DO: 使用相对路径 `<img src="logo.png" ... />`
3. ✅DO: 在浏览器中打开HTML，检查logo是否显示

**常见问题**：
- 问题：logo在浏览器中显示，但导出PNG时消失
- 原因：html2canvas无法渲染本地文件
- 解决：删除导出功能，让用户用浏览器"另存为"

### 3. 风格必须一致（定型法）

**产品一页纸和竞品对比必须用同一套CSS变量**：
- 相同的 `:root` 变量定义
- 相同的卡片样式
- 相同的字体规范
- 相同的间距和圆角

**检查方法**：
- 对比两个HTML文件的 `<style>` 部分
- 确认CSS变量定义一致

### 4. 内容必须有依据（接触法）

**禁止行为**：
- ❌ 编造brief中不存在的信息
- ❌ 使用拼音代替中文品牌名
- ❌ 主观评判竞品优劣（只做客观参数对比）
- ❌ **上下文污染：使用其他产品的信息**（如：把三伏贴的"郭昭明"用到休足贴）
- ❌ **信息遗漏：brief中有但没写进去**（如：注意事项少写1条）
- ❌ **信息理解错误：改写了技术参数**（如：pH≈8写成pH=8.0）
- ❌ **主观加工：添加个人理解**（如：客观描述改成主观评价）

**正确做法**：
- ✅ 所有内容从brief中提取
- ✅ 如果brief中没有相关信息，写`[待补充]`或询问用户
- ✅ 竞品对比只做客观参数对比
- ✅ **生成前，确认只读取了当前产品的brief**
- ✅ **提取后，逐条核对brief中的列表项数量（如：注意事项有几条）**
- ✅ **技术参数引用brief原文，不改写**

**上下文污染的常见表现**：
- 背书人/代言人错误（如：休足贴出现"郭昭明"）
- 技术名称错误（如：休足贴出现"冬病夏治"）
- 产品规格错误（如：把A产品的规格用到B产品）
- 古籍引用错误（如：把A产品引用的古籍用到B产品）

**信息遗漏的常见表现**：
- 注意事项少写几条（brief有9条，生成只有8条）
- FAQ少写几个（brief有6个，生成只有4个）
- 适用人群少写几类（brief有4类，生成只有3类）

**信息理解错误的常见表现**：
- 技术参数精度改变（如：≈8写成=8.0）
- 温度范围改变（如：-2℃写成-1~-3℃）
- 时间范围改变（如：3-4小时写成4小时）

### 5. 配色必须暖调（定值法）

**禁用冷色调**：
- ❌ 冷蓝色（如 `#4a90d9`）
- ❌ 冷绿色（如 `#2ecc71`）
- ❌ 冷灰色（如 `#95a5a6`）

**使用暖色调**：
- ✅ 暖墨绿：`#5a7a5a`
- ✅ 暖橄榄绿：`#6b8e6b`
- ✅ 暖棕色：`#8b6f47`

**竞品配色参考**（见 `references/brand_guidelines.md`）：
- 蔡氏福宁：品牌橙红 `#E85A24`
- 花王：暖墨绿 `#5a7a5a`
- 珍视明：暖橄榄绿 `#6b8e6b`
- 海氏海诺：暖棕色 `##8b6f47`

## 故障排除

### 问题1：品牌色显示不正确

**原因**：使用了错误的色值（如 `#87032`）

**解决**：
1. 打开 `references/brand_guidelines.md`，确认正确的色值（`#E85A24`）
2. 在HTML中搜索所有颜色相关的代码，替换为正确的色值
3. 重新预览HTML

### 问题2：Logo无法显示

**原因**：文件路径错误

**解决**：
1. 尝试方案A（相对路径）：
   ```html
   <img src="logo.png" alt="蔡氏福宁" />
   ```
2. 确认logo文件已复制到HTML同目录
3. 在浏览器中打开HTML，检查logo是否显示

### 问题3：配色不协调

**原因**：使用了冷色调

**解决**：
1. 打开 `references/brand_guidelines.md`，查看"竞品配色"表格
2. 确认所有颜色都在暖色相环内
3. 替换冷色调为对应的暖色调
4. 重新预览HTML

### 问题4：内容不准确

**原因**：编造了brief中不存在的信息

**解决**：
1. 打开brief文件，逐条核对信息图内容
2. 删除或修正编造的信息
3. 如果brief中没有相关信息，询问用户
4. 重新生成HTML
