## 产出物定义 📦

### 产品一页纸HTML
**文件路径**: `infographic/{产品名}-产品一页纸/{产品名}-产品一页纸.html`

**必须包含的板块**（按顺序）：
1. ✅ 顶部装饰条（6px渐变）
2. ✅ 头部区域（品牌logo + 产品名称 + 口号）
3. ✅ 核心卖点POD（2×2网格，4个卖点）
4. ✅ 产品信息（规格/使用方法/适用人群/禁忌人群，4列网格）
5. ✅ 注意事项（独立板块，2列列表，⚠图标）
6. ✅ 三种香型（如有，3列网格，绿/橙/黄渐变卡片）
7. ✅ 使用场景（3列网格）
8. ✅ 产品口播（浅橙红底+品牌色文字）
9. ✅ 常见问题FAQ（2列网格）
10. ✅ 底部区域（深色底+logo+免责声明）

**设计参数（定值法）**：
- 品牌色: `#E85A24`（PANTONE 1645C，✅DO NOT使用`#87032`等错误色值）
- Logo: 相对路径`logo.png`，CSS:`filter: brightness(0) invert(1)`（变白色）
- 背景: `#faf8f2`（宣纸暖白）
- 字体: 标题`Noto Serif SC`，正文`Noto Sans SC`
- 卡片: 白底+`#e0d5c0`边框+6px圆角+轻微阴影
- ✅DO NOT使用冷色调（蓝/冷绿/冷灰）

### 竞品对比HTML
**文件路径**: `infographic/{产品名}-竞品对比/{产品名}-竞品对比.html`

**必须包含的板块**：
1. ✅ 竞品定位（4张卡片，各用不同暖调背景）
2. ✅ 核心参数速查（4张参数卡片，通用化，不叫"技术路径"）
3. ✅ 多维度综合对比（雷达图三列排布：左右说明卡片 + 中间雷达图）
4. ✅ 核心结论（浅橙红底 + 品牌色标题）

**竞品配色（定值法）**：
- 蔡氏福宁: 品牌橙红 `#E85A24`
- 竞品1: 暖钢灰蓝 `#4a6b7a`
- 竞品2: 暖橄榄绿 `#5a8a5a`
- 竞品3: 暖铜棕 `#a07030`

### 中间文件（markdown）
**文件路径**: `infographic/{产品名}-{类型}/{产品名}_{类型}内容.md`

**用途**: 方便用户微调文案，再从模板生成HTML

## 工作流程

### 重要：使用模板生成HTML（不是重新设计）

**✅DO: 基于模板修改内容**
- ✅DO NOT重新写HTML结构——会导致设计风格走样（v5版本教训）
- ✅DO: 复制`assets/`目录中的模板到目标目录
- ✅DO: 读取markdown文件，提取内容填充模板占位符

**模板保证出品一致性**：
- 设计风格100%与v3版本一致
- CSS变量定义完全一致
- 板块顺序和布局完全一致

### 第一步：提取内容生成markdown

#### ☑SHOULD: 读取brief文件

**✅DO NOT编造内容**——所有内容必须从brief中提取。

1. **用户提供参考**：
   - 如果用户提供了brief文件路径，使用`Read`工具读取
   - 如果用户提供了IMA链接，使用`mcp__ima-mcp__fetch_media_content`读取

2. **用户未提供参考**：
   - ✅DO: 询问用户提供brief文件
   - ✅DO NOT: 自己编造产品信息

#### ☑SHOULD: 提取关键信息

按照`references/brief-structure.md`中的标准结构提取：

1. **产品层**：
   - 产品名称、规格、SKU列表
   - 核心卖点POD（特征/优势/利益/vs竞品/RTB证据）
   - 使用方法
   - 适用人群（从brief中提取，写关键内容）
   - 禁忌人群（从brief中提取，写关键内容）
   - 注意事项（从brief中提取，写关键内容，6-7条）
   
2. **心智层**：
   - 产品口号（slogan）
   - 口播文案（如有）
   
3. **市场层**：
   - 竞品信息（名称/价格/定位/技术路径）
   - 价格定位
   
4. **利益层**：
   - 目标人群
   - 使用场景
   - 核心价值

#### ☑SHOULD: 生成markdown文件

**✅DO: 严格按照以下规范生成markdown**：

1. **文件名**: `{产品名}_信息图内容.md`（或`{产品名}_竞品对比内容.md`）
2. **位置**: 与HTML信息图同目录（`infographic/{产品名}-{类型}/`）
3. **内容结构**: 严格按照`references/brief-structure.md`中的"信息图内容结构"
4. **✅DO NOT编造**: 如果brief中没有某个信息，写`[待补充]`而不是自己编

#### ☑SHOULD: 展示markdown文件给用户

1. 使用`present_files`展示markdown文件
2. 询问用户："文案内容是否需要微调？"
3. 如果用户提出修改，优先修改markdown文件（方便微调）
4. 修改完成后，再进入下一步
5. **⚠️ 雷达图确认（竞品对比专用）**：
   - 展示markdown后，额外询问用户：
     "雷达图维度和各品牌打分排名是否准确？请确认：\n     - 维度列表：{维度1}、{维度2}...\n     - 各品牌各维度排名：{品牌名} > {竞品1} > ..."
   - ✅DO：用户确认后，再写入HTML
   - ✅DO NOT：自己填打分数据（高频错误）

### 第二步：基于模板生成HTML（关键步骤）

#### ☑SHOULD: 复制模板到目标目录

**✅DO: 使用Bash工具复制模板**（不是重新写HTML）：

```bash
# 产品一页纸
cp assets/product-one-pager-template.html infographic/{产品名}-产品一页纸/{产品名}-产品一页纸.html

# 竞品对比
cp assets/competitor-comparison-template.html infographic/{产品名}-竞品对比/{产品名}-竞品对比.html
```

#### ☑SHOULD: 复制logo到HTML目录

```bash
cp /Users/zankus/标准文件/设计标准文件/蔡氏福宁无底色\ logo.png infographic/{产品名}-{类型}/logo.png
```

#### ☑SHOULD: 读取markdown，填充模板

1. 使用`Read`工具读取markdown文件
2. 解析markdown结构，提取各部分内容
3. 使用`Edit`工具替换HTML模板中的占位符：
   - `{产品名称}` → 产品名称
   - `{产品口号}` → 口号
   - `{POD1标题}` → POD1标题
   - ...（完整占位符列表见模板文件）

#### ✅DO: 关键设计规范检查（定型法）

**✅DO NOT跳过此检查**——品牌色错误是高频问题。

1. **品牌色检查**：
   - ✅DO: 在CSS中定义`:root { --brand: #E85A24; }`
   - ✅DO NOT: 使用`#87032`（错误色值，v3之前出现过）
   - ✅DO NOT: 使用`#FF0000`（纯红色，不是品牌色）

2. **Logo检查**：
   - ✅DO: 使用相对路径`<img src="logo.png" alt="蔡氏福宁" class="header-logo" />`
   - ✅DO: 确认logo文件已复制到HTML同目录
   - ✅DO: 在浏览器中打开HTML，检查logo是否显示

3. **配色检查**：
   - ✅DO: 所有辅助色都用暖调（见`references/brand_guidelines.md`）
   - ✅DO NOT: 使用冷蓝色（如`#4a90d9`）
   - ✅DO NOT: 使用冷绿色（如`#2ecc71`）

4. **内容检查**：
   - ✅DO: 所有内容从brief中提取
   - ✅DO NOT: 编造brief中不存在的信息
   - ✅DO: 品牌名用"蔡氏福宁"，✅DO NOT用拼音"CAISHIFUNING"

#### ☑SHOULD: 预览HTML文件

1. 使用`present_files`展示HTML文件
2. 在浏览器中打开，检查效果
3. 重点检查：
   - 品牌色是否正确显示
   - Logo是否正常显示
   - 配色是否和谐（全部暖调）
   - 内容是否准确（无编造）

#### ☑SHOULD: 根据反馈调整

1. 如果用户提出修改意见：
   - **优先修改markdown文件**（方便微调）
   - 修改完成后，重新从模板生成HTML（不要手动改HTML）
   
2. ✅DO NOT: 直接手动修改HTML文件——会导致下次生成时风格不一致

## 重要提醒

### 1. 品牌色必须准确（定值法）

**错误示例**：
- ❌ `#87032`（色值错误，v3之前出现过）
- ❌ `#FF0000`（纯红色，不是品牌色）
- ❌ `#E85A24`但写成了`#E85A24`（拼写错误）

**正确示例**：
- ✅ `#E85A24`（PANTONE 1645C 亮橙红）

**检查方法**：
- 在CSS中定义 `:root { --brand: #E85A24; }`
- 所有使用品牌色的地方用 `var(--brand)`，不要硬编码

### 2. Logo必须显示（接触法）

**检查清单**：
1. ✅DO: 确认logo文件已复制到HTML同目录
2. ✅DO: 使用相对路径 `<img src="logo.png" ... />`
3. ✅DO: 在浏览器中打开HTML，检查logo是否显示

**常见问题**：
- 问题：logo在浏览器中显示，但导出PNG时消失
- 原因：html2canvas无法渲染本地文件
- 解决：删除导出功能，让用户用浏览器"另存为"

### 3. 风格必须一致（定型法）

**产品一页纸和竞品对比必须用同一套CSS变量**：
- 相同的 `:root` 变量定义
- 相同的卡片样式
- 相同的字体规范
- 相同的间距和圆角

**检查方法**：
- 对比两个HTML文件的 `<style>` 部分
- 确认CSS变量定义一致

### 4. 内容必须有依据（接触法）

**禁止行为**：
- ❌ 编造brief中不存在的信息
- ❌ 使用拼音代替中文品牌名
- ❌ 主观评判竞品优劣（只做客观参数对比）

**正确做法**：
- ✅ 所有内容从brief中提取
- ✅ 如果brief中没有相关信息，写`[待补充]`或询问用户
- ✅ 竞品对比只做客观参数对比

### 5. 配色必须暖调（定值法）

**禁用冷色调**：
- ❌ 冷蓝色（如 `#4a90d9`）
- ❌ 冷绿色（如 `#2ecc71`）
- ❌ 冷灰色（如 `#95a5a6`）

**使用暖色调**：
- ✅ 暖墨绿：`#5a7a5a`
- ✅ 暖橄榄绿：`#6b8e6b`
- ✅ 暖棕色：`#8b6f47`

**竞品配色参考**（见 `references/brand_guidelines.md`）：
- 蔡氏福宁：品牌橙红 `#E85A24`
- 花王：暖墨绿 `#5a7a5a`
- 珍视明：暖橄榄绿 `#6b8e6b`
- 海氏海诺：暖棕色 `##8b6f47`

## 模板使用说明

**模板特点**（与v3版本完全一致）：
- 头部：渐变背景 + 右侧logo（filter变白色）+ brand-label
- Section标题：`::before` 伪元素竖线 + 底部边框
- POD卡片：暖纸背景 + 悬停效果 + 绝对定位半透明编号
- 产品信息板块：3列网格（规格/使用方法/注意事项）
- FAQ布局：2列网格
- 装饰元素：多处伪元素装饰圆
- 导出功能：已删除（用户要求）

**占位符列表**：
- `{产品名称}` - 替换为企业名称
- `{产品口号}` - 替换为企业口号
- `{POD1标题}` 到 `{POD4标题}` - 替换为核心卖点标题
- `{POD1特征}` 到 `{POD4特征}` - 替换为核心卖点描述
- `{规格列表}` - 替换为 `<li>` 标签列表
- `{使用方法列表}` - 替换为 `<li>` 标签列表
- `{注意事项列表}` - 替换为 `<li>` 标签列表
- `{香型1名称}` 到 `{香型3名称}` - 如果有香型则替换
- `{场景1标题}` 到 `{场景3标题}` - 替换为场景标题
- `{口播文案内容}` - 替换为口播文案
- `{FAQ列表}` - 替换为FAQ的HTML结构

### 竞品对比模板

**文件**：`assets/competitor-comparison-template.html`

**模板特点**（与产品一页纸风格完全一致，v5最终确认结构）：
- 头部：相同样式（渐变背景 + 右侧logo + brand-label）
- 竞品定位：4张卡片，各用不同暖调背景（brand/c1/c2/c3）
- 核心参数速查：4张参数卡片（通用化，不叫"技术路径"），卡片式布局
- 多维度综合对比：雷达图三列排布（左右说明卡片 + 中间雷达图）
  - 雷达图规范：蔡氏福宁fill opacity 0.13突出，竞品0.05；打分最高3.5/4（不要太满）；不显示刻度数字

## 故障排除

### 问题1：品牌色显示不正确

**原因**：使用了错误的色值（如 `#87032`）

**解决**：
1. 打开 `references/brand_guidelines.md`，确认正确的色值（`#E85A24`）
2. 在HTML中搜索所有颜色相关的代码，替换为正确的色值
3. 重新预览HTML

### 问题2：Logo无法显示

**原因**：文件路径错误

**解决**：
1. 尝试方案A（相对路径）：
   ```html
   <img src="logo.png" alt="蔡氏福宁" />
   ```
2. 确认logo文件已复制到HTML同目录
3. 在浏览器中打开HTML，检查logo是否显示

### 问题3：配色不协调

**原因**：使用了冷色调

**解决**：
1. 打开 `references/brand_guidelines.md`，查看"竞品配色"表格
2. 确认所有颜色都在暖色相环内
3. 替换冷色调为对应的暖色调
4. 重新预览HTML

### 问题4：内容不准确

**原因**：编造了brief中不存在的信息

**解决**：
1. 打开brief文件，逐条核对信息图内容
2. 删除或修正编造的信息
3. 如果brief中没有相关信息，询问用户
4. 重新生成HTML
