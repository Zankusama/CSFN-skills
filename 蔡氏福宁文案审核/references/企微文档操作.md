# 企微文档操作 — 白名单读取/写入模板

> 适用 skill：蔡氏福宁文案审核 v9+
> 企微 CLI 版本：0.1.9 | API：`wecom-cli doc`

---

## 一、读取白名单（步骤3a）

### 标准流程

```bash
# 1. 首次调用（不传 task_id）
wecom-cli doc get_doc_content \
  '{"docid":"dce_tRfn7KasEa5-jt0R9m9-VBhCrZtoyuoAyOFzwzwTdFt2vQ_WMSXYfU8HbfaCtB9h4DWuQtSz-1fzEcaUgAcw","type":2}'

# 2. 若 task_done=false，用返回的 task_id 轮询（最多5次）
wecom-cli doc get_doc_content \
  '{"docid":"dce_tRfn7KasEa5-jt0R9m9-VBhCrZtoyuoAyOFzwzwTdFt2vQ_WMSXYfU8HbfaCtB9h4DWuQtSz-1fzEcaUgAcw","type":2,"task_id":"<返回的task_id>"}'

# 3. task_done=true → content 字段即完整 Markdown
```

### 解析规则

白名单文档为**双轨格式**，AI 匹配以「🤖 AI 解析视图」段落的表格为准：

```
## 🤖 AI 解析视图（Skill 读取区）
| 词汇 | 背书依据 | 确认日期 |
|------|---------|----------|
| 本草润目 | 功效暗示，已法务背书，可用 | 2026-05-09 |
| 轻养 | 营销造词，多产品已长期使用... | 2026-06-12 |
```

提取方式：
1. 定位 `## 🤖 AI 解析视图` 标题
2. 解析其下方 markdown 表格（`| 词汇 | 背书依据 | 确认日期 |` 三列）
3. 匹配审核文案中的词汇

> ⚠️ 不直接解析 `📖 人工视图` 段（格式为 `- **词汇** · 日期 — 依据`），因为该格式无机器解析保证。

---

## 二、写入白名单（步骤4b）

### 双轨写入规则

新增词汇需**同步更新 3 个位置**：

| 位置 | 所在段落 | 格式 | 插入方式 |
|------|---------|------|---------|
| 人工视图 | `📖 人工视图` | `- **词汇** · YYYY-MM-DD — 依据` | 按日期倒序，插入到相同日期的词下方 |
| AI 解析表格 | `🤖 AI 解析视图` | `| 词汇 | 依据 | YYYY-MM-DD |` | 追加到表格末尾 |
| 维护记录 | `🕐 维护记录` | `| YYYY-MM-DD | 词汇名 | 触发场景 |` | 追加到记录表顶部（最新在上） |

### 完整 Node.js 模板

```javascript
const { execSync } = require("child_process");
const DOCID = "dce_tRfn7KasEa5-jt0R9m9-VBhCrZtoyuoAyOFzwzwTdFt2vQ_WMSXYfU8HbfaCtB9h4DWuQtSz-1fzEcaUgAcw";

// 新增词汇：[词汇, 背书依据, 确认日期, 触发场景]
const NEW_ENTRIES = [
  ["轻养", "营销造词，多产品已长期使用，组合词拆开单独可用，审核人2026-06-12确认", "2026-06-12", "线下展架文案审核"],
];

// ===== 1. 读取当前文档 =====
function readDoc() {
  let taskId = null;
  for (let i = 0; i < 5; i++) {
    const payload = taskId
      ? { docid: DOCID, type: 2, task_id: taskId }
      : { docid: DOCID, type: 2 };
    const res = JSON.parse(execSync(
      `wecom-cli doc get_doc_content '${JSON.stringify(payload)}'`,
      { encoding: "utf8" }
    ));
    const inner = JSON.parse(res.result.content[0].text);
    if (!inner.task_done) { taskId = inner.task_id; continue; }
    return inner.content;
  }
  throw new Error("读取失败");
}

// ===== 2. 拼接新内容 =====
function buildNewContent(oldContent, entries) {
  let content = oldContent;

  for (const [word, reason, date, scenario] of entries) {
    // 2a. 人工视图：按日期倒序插入
    const listSection = "## 📖 人工视图";
    const divider = "\n## 🤖 AI 解析视图";
    const listStart = content.indexOf(listSection);
    const listEnd = content.indexOf(divider, listStart);

    if (listStart >= 0 && listEnd >= 0) {
      const listPart = content.substring(listStart, listEnd);

      // 找到同日期（或更早）的最后一条，插入在其下方
      const datePattern = new RegExp(`- \\*\\*(.+?)\\*\\* · (${date})`, "g");
      const matches = [...listPart.matchAll(datePattern)];

      if (matches.length > 0) {
        // 在同日期的最后一条后插入
        const lastMatch = matches[matches.length - 1];
        const insertPos = listStart + lastMatch.index + lastMatch[0].length;
        const newLine = `\n- **${word}** · ${date} — ${reason}`;
        content = content.slice(0, insertPos) + newLine + content.slice(insertPos);
      } else {
        // 没有同日期条目，按日期插入正确位置
        const lines = listPart.split("\n");
        let inserted = false;
        for (let i = 0; i < lines.length; i++) {
          const line = lines[i];
          const m = line.match(/- \*\*.+?\*\* · (\d{4}-\d{2}-\d{2})/);
          if (m && m[1] < date) {
            const pos = listStart + lines.slice(0, i).join("\n").length;
            const newLine = `\n- **${word}** · ${date} — ${reason}`;
            content = content.slice(0, pos) + newLine + content.slice(pos);
            inserted = true;
            break;
          }
        }
        if (!inserted) {
          // 日期最新，插入到列表开头（第一条条目后面）
          const firstItemIdx = listPart.indexOf("\n- **");
          if (firstItemIdx >= 0) {
            const pos = listStart + firstItemIdx;
            const newLine = `\n- **${word}** · ${date} — ${reason}`;
            content = content.slice(0, pos) + newLine + content.slice(pos);
          }
        }
      }
    }

    // 2b. AI 解析表格：找到表格尾部追加新行
    const tableMarker = "## 🤖 AI 解析视图";
    const tableEndMarker = "\n\n---\n\n## 📋 维护说明";
    const tableStart = content.indexOf(tableMarker);
    const tableEnd = content.indexOf(tableEndMarker, tableStart);

    if (tableStart >= 0 && tableEnd >= 0) {
      const newRow = `\n| ${word} | ${reason} | ${date} |`;
      content = content.slice(0, tableEnd) + newRow + content.slice(tableEnd);
    }
  }

  // 2c. 维护记录表：在记录表顶部追加
  const recordMarker = "## 🕐 维护记录\n\n| 日期 | 录入词汇 | 触发场景 |\n|------|---------|----------|";
  const recordIdx = content.indexOf(recordMarker);
  if (recordIdx >= 0) {
    const insertAfter = recordIdx + recordMarker.length + 1;
    const newRecords = entries.map(([word, _, date, scenario]) =>
      `| ${date} | ${word}（${entries.length > 1 ? "等多条" : "1条"}） | ${scenario} |`
    ).join("\n");
    content = content.slice(0, insertAfter) + "\n" + newRecords + content.slice(insertAfter);
  }

  return content;
}

// ===== 3. 写入 =====
function writeDoc(content) {
  const payload = { docid: DOCID, content, content_type: 1 };
  const res = JSON.parse(execSync(
    `wecom-cli doc edit_doc_content '${JSON.stringify(payload)}'`,
    { encoding: "utf8" }
  ));
  const inner = JSON.parse(res.result.content[0].text);
  if (inner.errcode !== 0) {
    throw new Error(`写入失败 errcode=${inner.errcode} errmsg=${inner.errmsg}`);
  }
  console.log("✅ 写入成功");
}

// ===== 执行 =====
const oldContent = readDoc();
const newContent = buildNewContent(oldContent, NEW_ENTRIES);
writeDoc(newContent);
```

### 关键注意事项

1. **覆写模式**：`edit_doc_content` 是全量覆写，必须先读取原内容 → 拼接新行 → 整体写回
2. **必须用 Node.js**：Shell 拼接遇到特殊字符（引号/换行）会转义出错
3. **读取是异步**（需轮询 `task_done`），**写入是同步**（`errcode=0` 即成功）
4. **双轨同步**：人工视图 + AI 表格 + 维护记录 三个位置必须同时更新
5. **不要用 `\u000b`**：企微 CLI `content_type: 1` 写的是标准 Markdown，用 `\n` 换行即可
